## Docs Guide

This folder contains planning, implementation, and architecture documentation for Synapse.

### How to use these docs
- Start with **MVP focus** to understand current scope.
- Use **Backlog** to see what’s in progress vs. pending.
- Reference **Architecture/Tech** docs for system boundaries and design.
- Use **UI/API** docs for integration and frontend wiring.
- Check **Implementation status** for what’s shipped vs. remaining.

### Key documents
- **MVP focus**
  - `Implementation/MVP_STORY_DETAILING.md` — MVP scope and UI flow for Story Detailing.
  - `Implementation/IMPLEMENTATION_STATUS.md` — current implementation status summary.
- **Backlog & planning**
  - `Planning/SYNAPSE_BACKLOG.md` — prioritized backlog and status.
  - `Planning/USER_FLOW_STORY_DETAILING.md` — end‑to‑end user flows.
  - `Planning/UI_API_CONTRACTS.md` — UI ↔ API contract proposals.
  - `Planning/FIGMA_AGENT_MAPPING.md` — mapping between UI and agent flow.
  - `Planning/ROLES.md` — role definitions and access notes.
- **Architecture & tech**
  - `Planning/Tech/CURRENT_ARCHITECTURE.md` — current system layout.
  - `Planning/Tech/NORTH_STAR_ARCHITECTURE.md` — target architecture.
  - `agents_v1.0.md` — detailed agent architecture specification.
- **Deployment**
  - `DEPLOYMENT_VERCEL.md` — deployment notes for Vercel.

### Suggested reading order
1. `Implementation/MVP_STORY_DETAILING.md`
2. `Implementation/IMPLEMENTATION_STATUS.md`
3. `Planning/SYNAPSE_BACKLOG.md`
4. `Planning/Tech/CURRENT_ARCHITECTURE.md`
5. `Planning/Tech/NORTH_STAR_ARCHITECTURE.md`
